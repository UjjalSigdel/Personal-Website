# Roadmap

## Phase 1 — Cleanup

- Remove unused dependencies
- Remove dead code
- Remove unused components
- Improve folder organization
- Improve naming consistency

---

## Phase 2 — Architecture

- Improve component structure
- Improve code reuse
- Reduce complexity
- Improve maintainability

---

## Phase 3 — UI Refresh

Goals

- Move away from generic AI portfolio appearance
- Reduce reliance on purple gradients
- Improve typography
- Improve spacing
- Improve hierarchy
- Improve visual identity

---

## Phase 4 — Projects

Current project cards are limited.

Future improvements:

- Better cards
- Better project thumbnails
- Better filtering
- Better categorization

Introduce:

/projects

## Phase 5 — Performance

- Improve Lighthouse score
- Reduce bundle size
- Optimize assets
- Lazy loading where appropriate

---

## Phase 6 — Accessibility

- Keyboard navigation
- Better semantic HTML
- Better contrast
- Better screen reader support

---

## Phase 7 — SEO

- Better metadata
- Better Open Graph
- Better structured data
- Better search visibility

---

## Phase 8 — Future Features

Possible future additions:

- Blog
- Articles
- Certifications
- Project search
- Better analytics
- Interactive project timeline

Not all ideas need to be implemented.

Quality is preferred over quantity.

## Guiding Principles

Roadmap items are not fixed requirements.

The order may change as the project evolves.

Quality and maintainability always take priority over completing roadmap items quickly.
